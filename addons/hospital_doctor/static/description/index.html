Hospital Doctor

Consultation workspace and multi-branch outcome routing

Hospital Doctor is the clinical-core addon of the Hospital Management System suite, built on ``hospital_nurse``. It gives doctors a single-screen consultation workspace -- history, vitals, and allergy context alongside diagnosis entry and multi-select outcome routing -- plus a fast prescription builder.

Key Capabilities

  • Single-screen consultation: patient history, vitals, and an allergy banner alongside diagnosis entry.
  • Multi-select outcome routing: prescribe, lab, radiology, admit, and discharge can all be selected at once.
  • Extensible aggregate visit-state computation: a visit can sit in multiple department queues simultaneously, only reaching Billing once every branch is resolved.
  • Built-in prescription builder with medicine autocomplete and dosage/frequency/duration/route lines.
  • Same-day consultation amendment, fully audit-logged.
  • A 3-pane Doctor Dashboard: queue, consultation, and collapsible patient-history panel.
  • Prescription printout report.

Built for the Full Suite

Hospital Doctor defines the extension point (``hospital.visit._compute_pending_branches()``) that the future ``hospital_pharmacy``, ``hospital_lab``, ``hospital_radiology``, and ``hospital_ipd`` modules will hook into once built -- the routing logic is honest about what it can resolve today and clearly documents what each future module must add.
