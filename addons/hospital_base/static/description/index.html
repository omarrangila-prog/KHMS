Hospital Base

The foundation of a complete Hospital Management System

Hospital Base is the foundational layer of a complete, modular Hospital Management System built natively on Odoo 19. It provides the patient registry, the visit "spine" record that every clinical workflow attaches to, department and doctor master data, and a tamper-resistant audit trail -- all the building blocks the rest of the hospital_* suite (reception, nursing, doctor workspace, pharmacy, lab, radiology, IPD, dashboards) depends on.

Key Capabilities

  • Unique, sequence-generated patient and visit codes (PT/00001, VS/00001).
  • Non-blocking duplicate-patient detection by identity number.
  • Allergy and chronic-condition tracking on every patient record.
  • A full visit state machine: draft, waiting for nurse, waiting for doctor, in progress, admitted, billing, done, cancelled, void.
  • Department and doctor master data with weekly availability schedules.
  • An append-only, tamper-resistant audit log -- no group, including the administrator, can delete an audit entry.
  • Printable Patient ID Card report.
  • Multi-company record rules ready for hospital chains with multiple branches.

Built for the Full Suite

Hospital Base is designed to be extended, not duplicated. Later modules in the suite (Reception, Nurse Station, Doctor Workspace, Pharmacy, Laboratory, Radiology, IPD) build directly on top of the models defined here via standard Odoo inheritance, keeping the whole system consistent, auditable, and upgrade-safe.
