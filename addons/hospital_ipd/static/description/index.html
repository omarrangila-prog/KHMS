Hospital In-Patient Department

Full inpatient lifecycle for the Hospital Management System: ward/bed management, admission requests, bed assignment, mid-stay transfers, and discharge with length-of-stay billing.

Key Features

  • Ward and Bed master data with a real-time occupancy dashboard.
  • Admission state machine: Requested → Waiting for Bed → Admitted → Discharged.
  • Database-level partial unique index guaranteeing at most one active admission per bed, even under concurrent bed assignment attempts.
  • Mid-stay bed transfers recorded as a child of the same admission, never a new record.
  • Discharge blocked while any prescription/lab/radiology branch on the visit remains open.
  • Length-of-stay ward billing (days × daily rate), ready for a future billing module to post.
  • Discharge sub-types: Normal, Against Medical Advice, Deceased, Referred.
  • Ward-round nurse tasks linked to the admission, not just the OPD visit.
  • QWeb Discharge Summary PDF with take-home medication list.
  • Multi-company isolation via company record rules.

Dependencies

hospital_doctor hospital_nurse (bringing in hospital_reception, hospital_base).

Concurrency Guarantee

Two simultaneous admission attempts to the same bed: only one succeeds. The second fails cleanly at the database level via CREATE UNIQUE INDEX ... WHERE state = 'admitted', not a race-prone Python read-then-write check.
