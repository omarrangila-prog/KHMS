Hospital Security

Centralised security hardening for the Hospital Management System. Module 12/12 - the final layer applied on top of all other hospital_* modules.

Key Features

  • Cross-module multi-company isolation: every clinical model is scoped to the user's allowed companies via ir.rule.
  • Append-only audit log enforcement: no group (including Administrator) can delete hospital.audit.log entries.
  • Security Overview admin view: pre-filtered audit log with date and action presets, grouped by model.
  • Configuration → Security Overview menu (admin-only).
